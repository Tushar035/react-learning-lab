4.Events in React
-----------------
What Are Events in React? 

Events are actions triggered by users or the browser, such as:

 Clicking a button
 Typing in an input
 Submitting a form
 Moving the mouse
 Pressing a key

React handles events using camelCase names and functions.

Important Rules of React Events
---------------------------------
 Use camelCase ‚Üí onClick, onChange
 Pass function reference, not function call
 Events use SyntheticEvent
 Same event system works on all browsers

 1.Mouse Events
------------------
Used when user interacts with the mouse.

onClick
-----------
Triggered when an element is clicked.

<button onClick={() => alert("Clicked")}>Click Me</button>

Use case: Buttons, links, toggles

onDoubleClick
-------------
Triggered on double click.

<div onDoubleClick={() => alert("Double Clicked")}>
  Double Click Here
</div>

onMouseEnter
--------------
Triggered when mouse enters an element.

<div onMouseEnter={() => console.log("Mouse Entered")}>
  Hover Me
</div>

onMouseLeave
------------
Triggered when mouse leaves.

<div onMouseLeave={() => console.log("Mouse Left")}>
  Hover Area
</div>

onMouseMove
-------------
Triggered when mouse moves over element.

<div onMouseMove={() => console.log("Moving mouse")}>
  Move Mouse Here
</div>

onMouseDown / onMouseUp
-------------------------
Triggered when mouse button is pressed/released.

<button onMouseDown={() => console.log("Down")}
        onMouseUp={() => console.log("Up")}>
  Press
</button>

3Ô∏è.Form Events (VERY IMPORTANT)
------------------------------
Used in forms and inputs.

onChange
----------
Triggered when input value changes.

<input
  type="text"
  onChange={(e) => console.log(e.target.value)}
/>

Used for controlled components

onSubmit
----------
Triggered when form is submitted.

<form onSubmit={(e) => {
  e.preventDefault();
  alert("Form Submitted");
}}>
  <button type="submit">Submit</button>
</form>

onFocus
----------
Triggered when input gets focus.

<input onFocus={() => console.log("Focused")} />

 onBlur
-------
Triggered when input loses focus.

<input onBlur={() => console.log("Blurred")} />

onInput
-------------
Triggered immediately on input (less common).

<input onInput={() => console.log("Typing")} />

 üß™ Synthetic Event Object
----------------------------
React passes an event object automatically.
e is called as synthetic event over here
function handleClick(e) {
  console.log(e.type);
  console.log(e.target);
}

<button onClick={handleClick}>Click</button>

 Normalized across browsers
 Pooled for performance

Common Mistakes

Calling function directly  -- loads function on each render
------------------------
jsx
onClick={handleClick()}

Correct
------------
jsx
onClick={handleClick}

 Performance Tips
------------------------
 Avoid inline arrow functions in large lists
 Use useCallback for handlers
 Don‚Äôt overuse event listeners

InterviewReady Summary
------------------------------
 React events are handled using Synthetic Events with camelCase naming, allowing consistent and efficient handling of user interactions across browsers.
